enum Smartdb::SchedulingState

Each Operator takes 1 of the following states at a given time: WAITING: Child Operator does not even start outputting Records.

Initial state of non-leaf Operator. RUNNABLE: Child Operator already output at least a chunk of Records. But no Operator's instance is run on any worker. Initial state of leaf Operator. RUNNING: Only difference from RUNNABLE is RUNNING Operator has at least one instance run on any worker. FINISHED: Operator finished its execution.

Possible state transition: WAITING –> RUNNABLE <–> RUNNING –> FINISHED

Note
state is shared among all instances of each Operator.
